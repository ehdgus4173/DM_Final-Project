## 2. 다음 검토 계획

### 우선순위 1 — TEST R² 음수 원인 진단 ✅ **완료** (`LR/diag_test_r2.ipynb`)

**결론: 모델 결함이 아닌 temporal drift(타깃 분포 이동) — 버그가 아닌 발견.**

| 진단 | 결과 | 판정 |
|---|---|---|
| A. 분포 변화 | train 평균 12.73 → test 14.30 (+1.56), 표준편차 1.80→1.22 | **주범** |
| B. 미학습 카테고리 | only_test=∅, only_train=∅ | 무관 |
| C. 극단 잔차 | 상위 1% 제거 시 R² −1.28→−1.35 (악화), 잔차 상위 전부 양수 | 무관 (체계적 under-prediction) |
| α. 랜덤 vs 시간 | 랜덤 R²=**+0.116**(정상) vs 시간 R²=**−1.28** | **결정적 증거** |

→ 조치 확정: (a) 성능 지표는 **랜덤 split(+0.12) 병기**로 정상 베이스라인 확보, (b) 시간 split 음수는 4단계(ET 변환·드리프트) 스토리 근거로 보고. split 보정(카테고리 정합)은 불필요(B=∅).

### 우선순위 2 — VIF 처리 + 재적합 ✅ **완료** (`LR/refit_vif.ipynb`)
- **word_count 제거** → title_len VIF **7.77→1.09**, 전 변수 1.0~1.1대. 공선성 해소.
- **계수 안정**: 주요 변수 방향·유의성 유지(caps_ratio 0.43·has_bracket 0.39·exclaim_cnt 0.17·tag_cnt 0.02), title_len −0.014→−0.007(유의). 제거가 결론 불변.
- **성능**: 시간 split TEST −1.28 / 랜덤 split TEST **+0.115**. R² 손실 없음(0.113→0.112).
- **H4 재확인**: has_number(Music만 −0.47*)·exclaim_cnt(Music만 −) 부호 반전 유지 → H4 지지. 저장: `coef_comparison_top3_v2.csv`
- **확정 feature set**: title_len, caps_ratio, exclaim_cnt, question_cnt, has_number, has_bracket, tag_cnt.

### 우선순위 3 — 잔차 진단 마무리 ✅ **완료** (`LR/resid_diag.ipynb`)
- **등분산성(BP)** p≈8e-25 기각 / **정규성(JB)** p≈2e-34 기각 — 둘 다 n=4999 대표본 특성.
- 실질 크기는 경미: 왜도 **−0.39**, 첨도 **3.35**(정규 0/3에 근접), |z|>2 5.44%(기대 4.6%).
