**작성**: 2026-06-17, 발표 D-1
**입력**:

**산출 위치**: `Analysis3_channel/`
**용도**: 채널 트랙 착수 전 전체 맥락·노선 정리. 두 콘텐츠 트랙(제목·태그)이 왜 막혔는지 진단하고, 그 진단이 왜 "채널"을 가리키는지 기록. 실행 계획은 `docs/Analysis3_channel/phase1/plan.md`.

## 0. 한 줄 요약

> 제목(Analysis 1)·태그(Analysis 2) 두 콘텐츠 트랙이 **공통적으로** 낮은 설명력(LR R²≈0.12, 콘텐츠 피처 추가마다 ΔR²≈0)에 부딪혔다. 근본 원인은 "측정한 게 전부 **콘텐츠의 표면 속성**이었고, 정작 조회수를 만드는 **'누가 올렸나(채널)'를 한 번도 모델에 넣지 않았다**"는 것. 정제 과정에서 버려진 `channel_title`을 원본에서 복구해, 누수 없는 업로드시점 **채널 피처**(out-of-fold 평균 + 등장빈도)로 투입한다. 목적은 *"배경(채널) > 콘텐츠"를 숫자로 증명*하는 것. 그리고 **채널을 넣어도 분산 절반이 남으면, 그 잔여가 곧 Analysis 4(예측 시점 재프레이밍)의 근거**가 된다.

## 1. 현재 상황 (Analysis 1·2까지 확정)

### 1-1. 완료된 작업

| Analysis | 상태 | 핵심 산출물 |
|---|---|---|
| **EDA (Part A)** | ✅ 완료 | `dataset/cleaned_USvideos.csv`, H1/H2/H3, ET 변환 |
| **A1 — 제목 (LR+RF+감성)** | ✅ 완료 | `features_partB(_v2).csv`, OLS·VIF·잔차, VADER 실험, RF importance |
| **A2 — 태그 (단변량+다변량)** | ✅ 완료 | `step1~4` 태그효과, `step5` LR 통합 |

### 1-2. 핵심 발견 (콘텐츠 트랙 결산)

**A1 제목**
- 전체 LR rand test R² = **0.119**, 객관 제목지표 효과 약함
- 시간 split R² = **−1.28** → temporal drift (모델 결함 아닌 데이터 특성, 별개 finding)
- VADER 감성 추가 **ΔR² ≈ 0** (메타데이터 천장 낮음) — 단 카테고리별 부호 반전(H4)
- RF test R² = **0.245**(단 `trending_lag` 사후변수 포함 = Part B 기준 누수). importance에서 **업로드시점 변수 중 `tag_cnt`가 최강** → "개수가 이 정돈데 내용은?"

**A2 태그**
- 단변량: 카테고리별 흥행 태그 **완전 분리** (Music=pop↑·music/country/alternative↓, Film=disney/movie/trailer↑, Gaming=검출 0) → **H4 강력 지지**
- 다변량 통합: 카테고리·시간 통제 시 **ΔR² ≈ 0**, Film 태그 전부 소멸(카테고리 교란), Music만 약하게 생존 → "modest confirmatory null"

**관통 finding — H4(효과의 카테고리별 이질성)**: A1·A2 전반에서 "왜 pooled 예측이 안 오르나"의 원인으로 일관 등장. 별도 트랙이 아니라 **전 분석을 관통하는 발견**으로 유지.

## 2. 왜 Analysis 3(채널)로 가는가

### 2-1. 두 콘텐츠 트랙의 공통 한계

제목 객관지표 → 약함. 태그 내용 → ΔR²≈0. 감성 → ΔR²≈0.
세 번 모두 **"콘텐츠의 표면적 속성"** 을 측정했고, 세 번 모두 예측력을 거의 못 더했다.

### 2-2. 태그 트랙에서 진단한 직접 원인

1. **교란**: `disney`(단변량 ≈4.3배)는 태그 효과가 아니라 **카테고리·IP 브랜드 효과** — 통제하니 소멸
2. **선택효과**: 흔한 태그(music/country)가 음수 = "달면 망함"이 아니라 히트작이 전형적 태그를 회피
3. **이질성의 역설**: 우리가 계속 확인한 H4(카테고리별 부호 반전), **바로 그 이질성 때문에** 단일 pooled 모델에선 효과가 상쇄돼 R²가 안 오름

### 2-3. 근본 원인 (A1·A2 공통)

> **제목이든 태그든 감성이든, 우리가 측정한 건 전부 "콘텐츠가 무엇인가"였다.** 조회수를 실제로 만드는 건 콘텐츠 자체가 아니라, 그걸 **누가·어떤 도달 기반(채널 규모·팬덤·구독자)에서** 올렸는가다. 우리는 이 차원을 **한 번도 모델에 넣지 않았다.**
>
> 결정적 단서: RF에서 업로드시점 변수 중 최강이던 `tag_cnt`조차 다변량에서 '내용'으로 분해되지 않았다(A2 Step 5). 즉 그건 콘텐츠 신호가 아니라 **채널 운영 수준·제작 정성의 약한 프록시**였을 가능성이 크다 → **콘텐츠 너머(채널)를 가리키는 화살표.**

### 2-4. 데이터 마이닝 정합성

- `channel_title`은 정제 과정에서 단순 드롭됐을 뿐, **누수가 아니다**(업로드 시점에 당연히 아는 정보). 원본에서 복구는 정당.
- 구독자 수 직접값은 데이터셋에 없으나, **채널별 과거 평균 조회수가 채널 규모/팬덤의 프록시** 역할.
- out-of-fold target encoding = 인코딩 누수를 차단한 **고급 피처 엔지니어링** → 단순 회귀 프로젝트와의 차별화.

## 3. Analysis 3에서 무엇을 할 것인가

### 3-1. 가설

> **H_A3: 조회수는 콘텐츠(제목·태그)가 아니라 "누가 올렸는가(채널)"가 지배한다.**
> → 채널 정체성을 업로드시점 피처로 넣으면 R²가 크게 점프할 것이다. 이는 *"배경 > 콘텐츠"* 를 직접 증명한다.

### 3-2. 채널 피처 설계 (잠정)

1. **복구**: `dataset/USvideos.csv`에서 `video_id` 기준 `channel_title` 머지 (원본은 trending_date별 중복 → `video_id` dedup 후 머지). 정합성 확인.
2. **피처**:
   - 채널별 **out-of-fold 평균 `log_views`** (K-fold OOF target encoding — 같은 영상이 자기 타깃을 학습하는 누수 차단)
   - 채널 **등장 빈도**(트렌딩 횟수 = 채널 영향력 프록시)
   - **단일 등장 채널 fallback**: 카테고리 평균 또는 글로벌 평균으로 대체 (희소성 처리)
3. **누수 점검**: 채널 평균은 "업로드 전 채널의 과거 실적" 성격 → 업로드시점 정당. same-video leakage는 OOF로 차단. (이게 이 트랙의 핵심 방법론 포인트)

### 3-3. 비교 모델

- **M_content** = A1 확정 피처(`title_len, caps_ratio, exclaim_cnt, question_cnt, has_number, has_bracket, tag_cnt`) + `C(hour_bin)` + `C(publish_weekday)` + `C(category)` (현 베이스라인 ≈ rand R² 0.119)
- **M_content+channel** = M_content + 채널 피처
- 평가: rand/time split R² + K-fold CV R²(±std) + ΔR² (+ RF/GBM 병행으로 비선형 채널효과 확인 가능)

### 3-4. 누수 스코프 (비협상)

- **사용**: 콘텐츠 피처 + 카테고리/시간 + **채널 피처**(업로드시점)
- **미사용**: `trending_lag`, `days_on_trending`(사후), `likes`/`dislikes`/`comment_count`(참여도) → **Analysis 4에서 재프레이밍**

### 3-5. 명확히 안 하는 것

| 안 함 | 이유 |
|---|---|
| 구독자 수 직접 추정/외부 API 수집 | 데이터셋에 없음 — 한계로 명시 |
| 채널 임베딩/딥러닝 | 수업 범위 초과, 해석성 우선 |
| 참여·트렌딩 변수 투입 | 업로드시점 원칙 (A4로 이관) |

## 4. 발표 서사 통합 — 4-Analysis 스파인

```
[목표] 업로드시점 정보로 조회수를 예측한다
   ↓
[A1 제목] "제목 스타일이 좌우할 것이다"     → R²≈0.12, 약함 / RF서 tag_cnt 발견
   ↓
[A2 태그] "어떤 태그냐가 좌우할 것이다"      → ΔR²≈0 / 교란·선택효과·이질성의 역설
   ↓  (공통 진단: 우린 '콘텐츠 표면'만 봤다 → '누가 올렸나'를 안 봤다)
[A3 채널] "콘텐츠가 아니라 채널이 지배한다" ⭐ → R² 점프(예상 0.4~0.6), 그러나 분산 절반 잔존
   ↓  (소진: 최강 업로드시점 신호를 넣어도 천장)
[A4 재프레이밍] "정확도는 콘텐츠가 아니라 '예측 시점'의 함수다"
   ↓
[결론] 업로드시점 콘텐츠로는 조회수가 설명되지 않으며, 정확도는
       누가 올렸나(채널) > 무엇을 올렸나(콘텐츠) 순, 궁극적으론 '언제 예측하나'에 달려있다
```

- **관통**: H4(카테고리별 이질성)
- **A3의 위치**: 콘텐츠 표면의 한계를 **콘텐츠 너머(채널)로 정면 돌파**. 동시에 "그래도 절반의 천장"을 확인해 A4 피벗을 자연스럽게(소진으로) 도출.
- 분량: 슬라이드 2~3장, 발표 약 2~3분.

## 5. 정직성 플래그 (반드시 보고에 반영)

1. **피벗 근거는 "0.5는 나쁘다"가 아니다.** 0.4~0.6은 업로드시점치곤 사실 성공. 논리는 *"최강 업로드시점 신호(채널)를 넣어도 분산 절반이 남고, 그 나머지는 구조적으로 업로드시점엔 관측 불가(구독수·알고리즘·외부홍보가 데이터에 없음)거나 사후에만 알 수 있다 → 더 정확하려면 '무슨 콘텐츠 피처냐'가 아니라 '언제 예측하느냐'를 바꿔야 한다."* → 실제 숫자가 0.45든 0.6이든 **아크는 동일**, A4 문구만 결과 보고 확정.
2. **수정되는 건 "가설"이 아니라 "전제"다.** H1~H3는 틀린 게 아니라 차례로 답이 나온 것(제목 < 태그 < 채널). A4에서 바뀌는 건 *"업로드시점 정보로 정확한 예측이 가능하다"* 는 **연구의 전제(예측 시점)**. "가설이 순진했다"가 아니라 "정확한 예측의 조건을 재정의한다."
3. **target encoding 누수 주의**: 채널 평균을 같은 데이터로 만들어 같은 데이터에 쓰면 낙관 편향 → **OOF 필수**, test/CV 수치 중심 보고.
4. **채널 high-cardinality + 단일 등장 다수**: 일부 채널은 신호가 약함(fallback) → R² 기여는 반복 등장하는 대형 채널의 고조회수 꼬리에서 주로 발생함을 명시.
5. **temporal drift(time-split R²=−1.28)는 채널로 안 풀림** — 별개 finding으로 유지.
