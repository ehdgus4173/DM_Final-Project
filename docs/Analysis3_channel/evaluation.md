## 1. 무엇을 목표로 시작했나 (Goal)

### 1-1. 가설
> **H_A3: 조회수는 "무엇을 올렸나(콘텐츠)"가 아니라 "누가 올렸나(채널)"가 지배한다.**

### 1-2. 진입 동기 — 두 콘텐츠 트랙의 소진
제목(A1)·태그(A2)·감성이 **세 번 모두 ΔR²≈0**. 측정한 게 전부 "콘텐츠의 표면 속성"이었고, 조회수를 실제로 만드는 "도달 기반(채널 규모·팬덤)"을 **한 번도 모델에 넣지 않았다**는 진단. 결정적 단서: A1 RF에서 업로드시점 최강이던 `tag_cnt`가 다변량에서 '내용'으로 분해되지 않음 → 콘텐츠 너머(채널)를 가리키는 화살표.

### 1-3. 성공 기준 (착수 시점 설정)
- (1차) 채널 피처 투입 시 **ΔR²이 std 밖에서 유의하게 양수** — 콘텐츠 트랙의 ΔR²≈0과 질적으로 다른 결과
- (2차) `tag_cnt`가 채널 통제 후 약화 — "콘텐츠 신호 = 채널 프록시" 가설 검증
- (서사) R²가 점프하되 **천장이 남아** A4(예측 시점)로의 다리가 생길 것
- (방법론) **누수 없이** 달성 — 업로드시점 원칙 + OOF 인코딩 비협상

> ※ Background §5-1에서 R² 점프를 0.4~0.6으로 **예상**했으나, 이는 목표가 아니라 추정. 성공 기준은 절대 수치가 아니라 "ΔR²의 유의성 + 아크 성립"으로 설정됨.

## 2. 어떻게 진행했나 (Method)

| Step | 작업 | 핵심 결정 |
|---|---|---|
| 0 | `USvideos.csv`(원본)에서 `channel_title` 복구 → `video_id` dedup 후 `cleaned`에 머지 | 정제 과정에서 드롭된 것일 뿐 **누수 아님**(업로드시점 정보) |
| 1 | 채널 피처 OOF target encoding | **KFold(5) out-of-fold** 평균 — same-video 자기암기 누수 차단 / 단일·희소 채널 → 카테고리 평균 fallback |
| 2 | naive vs OOF 누수 시연 | naive(자기암기 포함) 동시 산출해 **낙관편향 정량화** |
| 3 | M_content vs M_content+channel | statsmodels OLS(계수·p) + sklearn KFold CV(±std) + rand/time split |
| 4 | RF/permutation importance 병행 | 비선형 채널효과 확인 |
| 5 | 발표 시각화 | R² 점프 + 잔여분산 |

**채널 피처 2종**: `chan_mean_oof`(채널 과거평균 log_views, OOF) · `chan_freq_log`(등장빈도 log1p, 영향력 프록시).
**누수 스코프(비협상)**: `trending_lag`/`days_on_trending`/참여지표 전면 배제 → A4 이관.

## 3. 결과는 무엇이었나 (Result)

### 3-1. 핵심 수치 (`step3_model_compare_channel.csv`)

| model | rand test R² | RMSE | time R² | CV R² (mean±std) |
|---|---|---|---|---|
| M_content | 0.1188 | 1.752 | −1.271 | 0.1067 ± 0.0168 |
| **M_content+chan (OOF)** | **0.2956** | 1.566 | −0.568 | **0.3229 ± 0.0207** |
| M_content+naive *(누수)* | 0.7366 | 0.958 | 0.362 | 0.7481 ± 0.0218 |

- **ΔR²(CV) = +0.216** (std≈0.02의 ~10배 → 견고). 콘텐츠 트랙 ΔR²≈0과 정반대 부호.
- 채널 계수 전부 유의: `chan_mean_oof` **+0.700**, `chan_freq_log` **+0.244** (p<.001).
- `tag_cnt` 계수 **반감**: 0.0218 → 0.0104.

### 3-2. RF / permutation importance (`step4_rf_importance.csv`)
- RF test R²: 0.112 → **0.304** (ΔR²=+0.192, LR과 일관 → 채널효과는 **선형 지배**).
- importance: **`chan_mean_oof` 0.525 (압도) > `chan_freq_log` 0.050 > `tag_cnt` 0.015**.
- `tag_cnt` 순위: M_content **1위 → +channel 3위**.

### 3-3. 채널 복구·인코딩 정합성
- 매칭률 **100%**(6249/6249), 고유 채널 2,100, **단일 등장 61.6%**, OOF fallback 23%.
- `chan_mean_oof`–`log_views` 상관 0.551 / `chan_mean_naive` 0.865.

## 4. 목표 대비 달성 / 미달성 (Assessment)

### 4-1. ✅ 달성한 것

1. **"배경 > 콘텐츠"를 수치로 입증.** ΔR²(CV)=+0.216, std 밖에서 견고. 1차 성공 기준 충족. 제목·태그·감성이 못 한 일을 채널 2개 피처가 해냄 → 앞선 세 트랙의 ΔR²≈0이 우연이 아니라 **구조적 이유(채널이 1차 결정자)였음을 사후 증명**.
2. **`tag_cnt` 정체 해소.** 채널 통제 후 계수 반감 + importance 1→3위. A1·A2를 관통하던 의문에 답. 2차 성공 기준 충족.
3. **누수 없이 달성.** OOF로 same-video 누수 차단, naive 대비 격차(+0.44)를 **시연 자료로 전환**. 방법론 기준 충족.
4. **A4로의 다리 확보.** R² 점프 + 천장 잔존이 동시에 성립 → 서사 기준 충족.

### 4-2. ❌ 달성하지 못한 것 (또는 한계로 드러난 것)

1. **R² 점프 폭이 예상(0.4~0.6)에 못 미침 (실측 0.30).** 단, 이는 "목표 미달"이 아니라 **예상 추정이 빗나간 것**. 원인은 §5에서 분해 — 핵심은 *채널이 약해서가 아니라 이 데이터로는 채널을 끝까지 잴 수 없어서*다.
2. **채널 신호의 대부분이 측정 불가.** 단일 등장 61.6% → OOF fallback 23%는 채널 평균이 아니라 카테고리 평균(기존 `C(category)` 더미와 중복) → 추가 설명력 ≈ 0. **채널 인코딩이 실제 작동하는 건 반복 등장 채널(~38%)뿐.**
3. **temporal drift 미해결.** time-split R²: −1.27 → −0.57. 개선됐으나 여전히 음수. 채널로 안 풀림 → 별개 finding 유지.
4. **`tag_cnt` "프록시" 단정 불가.** 반감(0.0218→0.0104)이지 소멸이 아님 → 순수 콘텐츠 성분 일부 잔존 가능. 발표 표현은 **"채널로 상당 부분 흡수됐다"**가 정직 (≠"순수 프록시였다").

## 5. 결론적으로 어떤 정보를 얻었나 (What we learned)

### 5-1. 본체 결론 (A3의 메인 기여)
조회수의 일차 결정자는 콘텐츠 표면이 아니라 **"누가 올렸나"**다. 그리고 콘텐츠 신호처럼 보이던 `tag_cnt`조차 상당 부분이 **채널 운영수준의 프록시**였다.

### 5-2. "왜 70%가 남나"의 정직한 분해 ⭐
잔여 분산 0.70은 단일 원인이 아니라 **세 겹**이다. 이 분해가 A3 평가의 핵심 통찰이며 A4 정당화의 토대다.

| 구성 | 성격 | 함의 |
|---|---|---|
| **(1) 채널 sparsity** | 61.6%가 단일 등장 → 23% fallback은 카테고리 평균 ≈ 기존 더미와 중복 | "채널이 약하다"가 아니라 **이 데이터로 채널을 끝까지 못 잰다**(데이터 한계) |
| **(2) survivorship** | 데이터셋 = **이미 트렌딩에 오른 영상만**. 채널이 가장 잘 설명할 변동("뜬다 vs 망한다")이 표본 진입 단계에서 이미 소거됨 | 어떤 방법을 써도 R²가 구조적으로 눌림(데이터 한계) |
| **(3) 진짜 관측 불가** | 구독자 수·추천 알고리즘·외부 홍보·바이럴 운 → 업로드시점 데이터에 부재 | (a) 어차피 데이터에 없음 / (b) 사후에만 관측 가능 |

**naive 0.74의 정체**: "놓친 천장"이 아니다. 단일 등장 채널의 naive 평균 = 자기 영상 log_views = **자기 정답 암기**(예: GoDucksdotcom의 naive=6.558=자기 log_views). 0.74와 0.30의 격차는 잃어버린 신호가 아니라 **걷어낸 환상**이다.

→ 따라서 **0.30은 기대 이하가 아니라 이 데이터 구조의 천장에 가깝다.** 발표 프레이밍: "채널 효과가 작더라"(✗) → **"채널이 더 클 텐데 이 데이터로는 여기까지만 잰다"**(✓).

## 6. 그래서 A4 접근을 어떻게 결정했나 (Decision → A4)

### 6-1. A3가 A4에 넘기는 것
잔여 70%를 §5-2처럼 분해하면 A4의 정당한 표적이 명확해진다:
- (1) sparsity·(2) survivorship·(3a) 관측불가 → **데이터의 구조적 한계** (정직하게 "여기까진 못 잰다"로 명시, A4가 "풀" 대상이 아님)
- **(3b) 사후에만 관측 가능한 정보**(trending_lag·초기 참여도) → **이것만이 A4의 정당한 동력**

### 6-2. A4의 핵심 논리 (수정 확정)
> "70%가 안 풀린다 → 예측 시점을 바꾼다"는 자동으로 따라오지 **않는다.** A4의 정확한 주장은:
> **"업로드시점 최강 신호(채널)를 넣어도 분산 절반 이상이 남고, 그 잔여 중 구조적 한계분을 걷어내면 남는 핵심은 *사후에만 알 수 있는 정보*다. 따라서 더 정확하려면 '무슨 콘텐츠 피처냐'가 아니라 '언제 예측하느냐'를 바꿔야 한다."**

### 6-3. A4가 정식 해소할 모순 4종 (A3에서 노출)
1. 잔여 분산 70% — §5-2 분해로 "한계분 vs 사후정보분" 분리
2. temporal drift (time R² 여전히 음수) — 예측 시점 재정의로 흡수
3. naive vs OOF 격차(+0.44) — 누수 시연, OOF만 헤드라인
4. RF 헤드라인 R²=0.245가 `trending_lag`(사후변수) 포함값 → 누수 서사와 모순 → 예측 시점 명시로 정식 해소

### 6-4. 서사적 정당성 (스코프 확대의 자기증명)
title → tag → channel로 **스코프를 세 번 능동적으로 확대**한 과정 자체가 "콘텐츠로는 안 된다"의 귀납적 증거. A4의 예측 시점 피벗은 *갑작스러운 전환*이 아니라 **이 확대 과정의 자연스러운 종착점**이다. 바뀌는 것은 "가설"이 아니라 연구의 **전제(예측 시점)**.
