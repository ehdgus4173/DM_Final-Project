## 0. 한 줄 요약

> 제목(A1)·태그(A2)·채널(A3)로 스코프를 세 번 확대했고, 최강 업로드시점 신호(채널)를 넣어도 **분산 70%가 남았다.** A4는 이 70%를 **풀려고 하지 않는다.** 대신 70%를 *구조적 한계분*과 *사후에만 관측 가능한 정보분*으로 분해하고, **예측 시점을 업로드 → 트렌딩 직후 → 참여 발생 후로 뒤로 옮기면 R²가 단조 상승**함을 보인다. 결론: **정확도는 "무슨 콘텐츠 피처냐"가 아니라 "언제 예측하느냐"의 함수다.** 이로써 연구의 *가설*이 아니라 *전제*("업로드시점 정보로 정확히 예측 가능하다")가 재정의된다.

## 2. 왜 A4(예측 시점)로 가는가

### 2-1. 스코프 확대의 귀납적 종착점
title → tag → channel로 **세 번 능동적으로 스코프를 확대**한 과정 자체가 "업로드시점 콘텐츠/배경 정보로는 한계가 있다"의 누적 증거다. A4는 갑작스러운 전환이 아니라 **이 확대의 자연스러운 끝**: 더 넓힐 콘텐츠/채널 차원이 소진됐다면, 남은 자유도는 **"무엇을 보느냐"가 아니라 "언제 보느냐"**뿐이다.

### 2-2. A3가 노출한 4개 모순 — A4가 정식 해소
1. **잔여 분산 70%**: §1-2 분해로 "한계분 vs 사후정보분" 분리.
2. **temporal drift** (time-split R²: A1 −1.27 → A3 −0.57, 여전히 음수): 세계가 시간에 따라 변한다는 직접 증거 → "예측 시점" 주제와 동일 축. A4에서 별개 결함이 아니라 **시점 의존성의 한 증상**으로 흡수.
3. **naive vs OOF 격차(+0.44)**: 누수 시연. A4의 사후정보 모델도 동일하게 "누수임을 명시한 시연"으로 다룸 — 헤드라인 예측치는 여전히 OOF 0.30.
4. **RF 헤드라인 R²=0.245가 `trending_lag`(사후변수) 포함값**: A1~A3 누수 원칙과 모순. → A4가 **"그건 업로드시점 예측이 아니라 트렌딩-직후 예측이었다"**로 재라벨링하여 정식 해소. 모순이 아니라 **다른 예측 시점의 결과**.

### 2-3. 데이터 마이닝 정합성
단순히 변수를 더 넣어 R²를 올리는 게 아니라, **누수 경계 = 예측 시점**이라는 프레임으로 "각 시점에서 무엇을 알 수 있고 정확도가 어떻게 변하는가"를 구조화 → 데이터 마이닝의 핵심(언제·무엇이 관측 가능한지에 따른 예측 가능성 분석)과 일치. `trending_lag`를 "누수"가 아니라 **"트렌딩 시점 정보"**로 재해석하는 것이 차별화 포인트.

## 3. A4에서 무엇을 할 것인가

### 3-1. 핵심 가설 (전제의 재정의)

> **H_A4: 예측 정확도는 콘텐츠/채널 피처의 종류가 아니라 "예측 시점"의 단조 함수다.**
> 예측 시점을 업로드 → 트렌딩 직후 → 참여 발생 후로 옮길수록 R²는 계단식으로 상승한다.

수정되는 것은 H1~H3(틀린 게 아니라 차례로 답이 나옴: 제목<태그<채널)이 아니라, 연구의 **전제** — *"업로드시점 정보만으로 정확한 예측이 가능하다"*. A4는 이 전제를 **"정확한 예측의 조건은 예측 시점이다"**로 재정의한다.

### 3-2. 예측 시점 사다리 (4단 — `r2_ceiling_test.py` 확장)

기존 하니스는 S0/S1/S2 3단이고 **채널 단이 빠져 있다.** A4는 A3 채널 피처를 끼워 **4단 사다리**로 재구성:

| 시점(rung) | 추가 정보 | 관측 가능 시점 | 예상 rand R² |
|---|---|---|---|
| **T0 콘텐츠** | 제목·태그·시간·카테고리 | 업로드 직전 | ~0.12 (A1) |
| **T1 +채널** | `chan_mean_oof`, `chan_freq_log` | 업로드 직전 (과거실적) | ~0.30 (A3) |
| **T2 +트렌딩 메타** | `trending_lag`, `days_on_trending` | **트렌딩 진입 후** | (실측) |
| **T3 +참여도** | `log_likes`, `log_comment_count` 등 | **참여 발생 후** | (실측) |

- T0~T1 = 업로드시점(누수 없음, 헤드라인). T2~T3 = 사후시점(**의도된 누수, "다른 예측 시점"으로 명시**).
- 같은 split·같은 모델(LR/RF/GBM)로 일관 평가. **단조 상승 곡선 = "WHEN not WHAT"의 시각적 증명.**
- `r2_ceiling_test.py`에 T1(채널) 단 추가 + 4단 출력으로 확장. 입력은 `cleaned_USvideos.csv`(사후변수 포함) + `step1_channel_features.csv` 머지.

### 3-3. 잔여분산 분해 (선택, 시간 허용 시)
T1(업로드시점 천장 0.30)에서 남는 0.70을 §1-2 표로 정량 제시: T2~T3이 회수하는 분산(=사후정보분 3b) vs 끝까지 안 풀리는 분산(=구조적 한계분 1·2·3a). "사다리를 끝까지 올라도 안 풀리는 바닥"을 보여 정직성 확보.

### 3-4. 명확히 안 하는 것

| 안 함 | 이유 |
|---|---|
| 사후정보 모델 R²을 "우리의 예측 성과"로 주장 | **누수임을 명시한 시연** — 시점 논제 증명용이지 성과 아님 |
| 70% 전부를 "풀어내려" 시도 | (1)(2)(3a)는 데이터 한계 — 정직하게 남김 |
| 외부 데이터(구독자수 API 등) 수집 | 범위 초과, 한계로 명시 |
| 새 모델 알고리즘 도입 | 기존 LR/RF/GBM 일관 사용 (시점 비교가 목적) |

## 4. 발표 서사 통합 — 스파인 완성 (결론 챕터)

```
[목표] 업로드시점 정보로 조회수를 예측한다
   ↓
[A1 제목]  R²≈0.12, 약함 / RF서 tag_cnt 발견
   ↓
[A2 태그]  ΔR²≈0 / 교란·선택효과·이질성
   ↓
[A3 채널]  R² 0.30으로 점프(배경>콘텐츠) ⭐, 그러나 70% 잔존
   ↓  (소진: 최강 업로드시점 신호도 천장)
[A4 재프레이밍] "정확도는 콘텐츠가 아니라 '예측 시점'의 함수다" ⭐⭐
   - 예측 시점 사다리: 0.12 → 0.30 → (T2) → (T3) 단조 상승
   - 70% 분해: 사후정보분(시점으로 회수) vs 구조적 한계분(끝내 잔존)
   ↓
[결론] 업로드시점 콘텐츠로는 조회수가 설명되지 않으며,
       정확도는 누가>무엇 순, 궁극적으론 '언제 예측하나'에 달려있다.
       trending_lag는 누수가 아니라 "트렌딩 시점 정보"다.
```

- **관통 finding**: H4(카테고리 이질성) — A4는 pooled 진행하나 시점 사다리에 카테고리 통제 유지.
- **A4의 위치**: 스파인의 **결론**. A3의 "천장"을 "시점의 함수"로 승화 → 4개 모순 동시 해소.
- 분량: 슬라이드 2~3장(사다리 곡선 + 70% 분해), 발표 약 2~3분.

## 5. 정직성 플래그 (반드시 보고에 반영)

1. **사후정보 모델은 성과가 아니라 시연이다.** T2~T3의 높은 R²은 "우리가 잘 예측했다"가 아니라 "**더 나중에 보면 당연히 더 잘 보인다**"는 시점 논제의 증명. 헤드라인 업로드시점 예측치는 OOF 0.30 고정.
2. **A4는 70%를 "풀지" 않는다.** (1)sparsity·(2)survivorship·(3a)관측불가는 데이터 한계로 남긴다. A4가 회수하는 건 (3b)뿐. "정확도가 오른다"가 아니라 "**관측 시점이 늦어져서** 오른다"가 정확한 표현.
3. **전제 재정의 ≠ 가설 기각.** H1~H3는 차례로 답이 나온 것(제목<태그<채널). 바뀌는 건 "정확한 예측이 가능하다"는 **전제(예측 시점)**. "가설이 순진했다"가 아니라 "정확한 예측의 조건을 재정의한다."
4. **temporal drift는 시점 논제와 한 축이되 별개 증상.** time-split 음수 R²은 "세계가 변한다 → 과거로 미래를 맞추기 어렵다"로 시점 의존성의 한 형태. 단, sparsity·survivorship과는 다른 원인이므로 뭉뚱그리지 않음.
5. **RF R²=0.245 재라벨링은 팀 합의 필요.** Phase 4 머지 전 "그 수치는 T2(트렌딩 직후) 예측"임을 팀과 통일. 누수 경계 미해결 이슈의 정식 종결점.
